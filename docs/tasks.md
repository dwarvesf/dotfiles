- [x] S-64: Continuous dotfiles watcher (live -> repo working tree). Two LaunchAgents -- launchd `WatchPaths` (regenerated from `chezmoi managed`, ~120 leaves) for native zero-process coverage of every tracked file's mtime, plus `fswatch -r` daemon on `~/.claude` / `~/.config/zed` / `~/.config/fish` for recursive coverage including subdirs created since the last apply. Both pipe into mkdir-locked `dotfiles-watcher-tick` which sleeps 2s (debounces editor-save bursts) then loops `chezmoi re-add` on every `^.M ` row in `chezmoi status` up to 3 passes. Working-tree only: no auto-commit, no auto-push (operator reviews via `git status`). New verb `dotfiles watch {install|uninstall|status|now|tail}`. Distinct from `/dotfiles-sync` -- the watcher is the "absorb drift on already-managed files" slice, so /dotfiles-sync's drift section stays empty and the skill's report focuses on classifying new things and committing. Self-contained test runner at `tests/dotfiles-watch.sh` covers 10 cases (shellcheck both wrappers, render+lint two templates, no-op-when-clean, single-pass absorb, drift-loop iteration, mkdir-lock coalescing, MM-row absorb, Brewfile entry); 10/10 pass on Mac mini. Implementation: `home/dot_local/bin/dotfiles-watcher-{tick,fswatch}`, `home/Library/LaunchAgents/com.truonghan.dotfiles-watcher-fswatch.plist.tmpl`, `home/.chezmoiscripts/run_onchange_after_dotfiles-watcher.sh.tmpl`. Original regex `^ M ` only caught dest-only-modified rows and missed real-world `MM` (both-changed) rows; widened to `^.M ` during deploy on Mac mini after live `~/.tool-versions` edit reported `MM` and the watcher exited silently. End-to-end demo on Mac mini 2026-05-12: edit absorbed in <8s, sed-revert absorbed on next tick, source returns to HEAD. User-facing docs in `docs/guide.md` § 4 "The background watcher."
